My opinion:

Q: What factors determine whether a programming language is secure or not?

A: A programming language able to prevent any cyber security vulnerability, for example, buffer overflows, input validation issues, access control issues, cryptographic weaknesses, information leaks, insecure file operations, etc. A secured programming language should be able to hash the password and data, so that they wonâ€™t be transferred as plain text over a public network (Pillai, 2017).

Q: Could Python be classed as a secure language? Justify your answer.

A: It can. Because the newest version of Python has solved many security buys from the previous version, to prevent the possibility of cyber-attack. Also, it has libraries for encryption and hashing, to prevent the passwords and data being transferred as plain text, reduce the risk of sniffing (Pillai, 2017).

Q: Python would be a better language to create operating systems than C. Discuss

A: It depends compare with Python, C cannot provide some new functionality, such as code reusability and OOP concepts (PSTAnalytics, 2020); but on the other hand, as Python is a very high-level programming language. We still need to rely on C on some low-level programming when trying to build an OS with Python. Also, Python needs longer CPU time for interpretation, compare with C, which will effect the OS performance if it is written by Python (Utkarsh, 2021).
